Tech Stack

Comprehensive overview of the technologies used in the TuPhung Project.

Overview

The TuPhung Project is built using a modern tech stack that combines the best tools and frameworks for building scalable, maintainable, and feature-rich web applications. The stack is divided into frontend, backend, database, and DevOps components.

Database

Primary Database

PostgreSQL is used as the primary relational database for storing structured data such as user accounts, tasks, events, and other application data.

  • Version: 15.3
  • Features Used: JSON data types, full-text search, triggers, stored procedures
  • Extensions: pgcrypto, uuid-ossp, pg_trgm

Caching and Session Storage

Redis is used for caching frequently accessed data, storing session information, and supporting real-time features like chat and notifications.

  • Version: 7.0.12
  • Features Used: Key-value storage, pub/sub, sorted sets, expiration policies

DevOps and Infrastructure

Containerization

Docker is used for containerizing the application, ensuring consistent environments across development, testing, and production.

  • Docker: 24.0.2
  • Docker Compose: 2.19.1

CI/CD

GitHub Actions is used for continuous integration and deployment, automating the build, test, and deployment processes.

  • Build Workflow: Builds and tests the application on every push and pull request
  • Deployment Workflow: Deploys the application to staging and production environments
  • Code Quality Workflow: Runs code quality checks and security scans

Cloud Infrastructure

AWS is used for hosting the application and related services.

  • EC2: Application hosting
  • RDS: PostgreSQL database
  • ElastiCache: Redis caching
  • S3: File storage
  • CloudFront: Content delivery
  • Route 53: DNS management

Local Development Setup

The project includes a Docker Compose configuration for setting up the local development environment. This includes PostgreSQL, Redis, and other required services. 

# docker-compose.yml (excerpt)
version: '3.8'

services:
  postgres:
    image: postgres:15.3
    environment:
      POSTGRES_USER: tuphung
      POSTGRES_PASSWORD: password
      POSTGRES_DB: tuphung_db
    ports:
      - "5432:5432"
    volumes:
      - postgres-data:/var/lib/postgresql/data

  redis:
    image: redis:7.0.12
    ports:
      - "6379:6379"
    volumes:
      - redis-data:/data

volumes:
  postgres-data:
  redis-data:
