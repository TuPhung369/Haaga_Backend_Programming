Calendar System

Comprehensive documentation of the Calendar implementation in the TuPhung Project.

Overview

The Calendar system in the TuPhung Project provides a comprehensive event management solution that allows users to schedule, view, and manage events. It supports different calendar views (month, week, day), event categories, recurring events, reminders, and sharing capabilities.

Key Features

  • Multiple Calendar Views - Month, week, day, and agenda views
  • Event Management - Create, edit, delete, and view event details
  • Recurring Events - Support for daily, weekly, monthly, and yearly recurring events
  • Event Categories - Color-coded categories for different types of events
  • Event Reminders - Notifications for upcoming events
  • Calendar Sharing - Share calendars with other users
  • Drag and Drop - Easily reschedule events by dragging
  • Event Search - Search for events by title, description, or category
  • Integration with Tasks - Link events to tasks in the Kanban board

Frontend Implementation

The Calendar frontend is built using React with TypeScript and leverages the react-big-calendar library for the calendar interface. The state management is handled through Redux, with a dedicated slice for calendar-related state.

Calendar Component

The main Calendar component provides a comprehensive interface for viewing and managing events. It supports multiple views (month, week, day, agenda) and allows users to create, edit, and delete events.

Key features of the Calendar component:

  • Multiple view options (month, week, day, agenda)
  • Event creation through clicking on time slots
  • Event editing through clicking on existing events
  • Color-coded events based on category
  • Drag and drop for rescheduling events

Event Management

The Calendar system provides comprehensive event management capabilities, including creating, editing, and deleting events. Events can have various properties such as title, description, date/time, category, location, and recurrence pattern.

Key event handlers:

  • handleSelectSlot - Triggered when a user selects a time slot to create a new event
  • handleSelectEvent - Triggered when a user clicks on an existing event to edit it
  • handleFormSubmit - Handles the submission of the event creation/editing form
  • handleDeleteEvent - Handles the deletion of an event

Example of event creation and editing:

// Event form submission handler
const handleFormSubmit = async () => {
  try {
    const values = await form.validateFields();
    
    const eventData = {
      id: selectedEvent?.id || '',
      title: values.title,
      description: values.description,
      start: values.dateRange[0].toDate(),
      end: values.dateRange[1].toDate(),
      allDay: values.allDay,
      category: values.category,
      location: values.location,
      recurring: values.recurring,
      recurrencePattern: values.recurring ? values.recurrencePattern : null,
      reminder: values.reminder,
      createdBy: user?.id || ''
    };
    
    if (isEditMode && selectedEvent) {
      // Update existing event
      const updatedEvent = await updateEvent(selectedEvent.id, eventData);
      dispatch(updateEventAction(updatedEvent));
      message.success('Event updated successfully');
    } else {
      // Create new event
      const newEvent = await createEvent(eventData);
      dispatch(addEvent(newEvent));
      message.success('Event created successfully');
    }
    
    setIsModalVisible(false);
    form.resetFields();
  } catch (error) {
    console.error('Failed to save event:', error);
    message.error('Failed to save event');
  }
};
